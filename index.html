Investigación y aplicación de metodologías BEM y camelCase

¿Qué Son?

Las normas BEM (Block, Element, Modifier) y CamelCase son dos convenciones de nomenclatura que se utilizan en el desarrollo web, especialmente en CSS. La BEM es una metodología para organizar clases CSS de forma modular, mientras que el CamelCase es una forma de escribir nombres de variables, funciones, etc. donde la primera letra de cada palabra (excepto la primera) se escribe en mayúscula, sin espacios.

BEM (Block, Element, Modifier):

  • Block: Un componente independiente reutilizable, como un formulario o un botón.
  • Element: Una parte interna de un bloque, como el título de un formulario o el icono de un botón.
  • Modifier: Una clase que modifica el estilo o el comportamiento de un bloque o un elemento.

El camelCase:

  • Es una convención de escritura que une dos o más palabras sin espacios, utilizando la primera letra de cada palabra en mayúscula, excepto la primera, que comienza con minúscula.
  • Se usa comúnmente para nombrar variables, funciones y otras entidades en lenguajes de programación.

¿Para Que Sirven?

► BEM (Block, Element, Modifier) sirve para:

  • Organizar el código CSS de forma modular y reutilizable.
  • Identificar fácilmente a qué parte del diseño corresponde cada clase.
  • Evitar conflictos de nombres en CSS (por ejemplo, que dos .titulo diferentes se mezclen).
  • Facilitar el mantenimiento y escalabilidad de proyectos grandes.

► El camelCase sirve para:

  • Mejora la legibilidad: Permite identificar las palabras individuales dentro de un nombre, facilitando su comprensión.
  • Es una convención común: Muchos lenguajes de programación y frameworks web la utilizan.
  • Facilita la escritura: Evita la necesidad de usar guiones bajos o espacios para separar palabras.

Como Usarlos Correctamente

► BEM se basa en tres componentes:

  • Bloque (Block):

    • Representa un componente independiente de la interfaz de usuario, como un formulario, un menú de navegación, una tarjeta, etc. Los bloques suelen tener un nombre descriptivo que refleja su función, como form, navigation, o card.

  • Elemento (Element):

    • Es un componente interno de un bloque, como un título dentro de una tarjeta, o el input de un campo de texto en un formulario. Los elementos se nombran utilizando el nombre del bloque, seguido de dos guiones bajos (__) y el nombre del elemento, por ejemplo, card__title o form__input.

  • Modificador (Modifier):

    • Permite modificar la apariencia o el comportamiento de un bloque o elemento sin duplicar el código CSS. Se utiliza para crear variantes de un bloque o elemento, como un botón principal, un botón secundario o un botón desactivado. Los modificadores se nombran utilizando el nombre del bloque o elemento, seguido de dos guiones (--) y el nombre del modificador, por ejemplo, button--primary o form__input--error.

► CamelCase es una convención para nombrar variables, funciones y métodos en programación, donde la primera letra de cada palabra (excepto la primera) se escribe en mayúscula y las palabras se concatenan sin espacios.

En detalle:

  • Primer palabra:La primera palabra se escribe con la primera letra en minúscula y el resto de las letras en minúscula.
  • Palabras siguientes:La primera letra de cada palabra subsiguiente se escribe en mayúscula.
  • Sin espacios: Las palabras se unen directamente sin espacios ni guiones bajos.
